## Overview
      Wadda is a Minimum Viable Product (MVP) project that aims to build a website offering digital invitation design services. The project focuses on developing an intuitive and user-friendly website UI/UX design to provide the best experience for potential users in choosing and ordering digital invitations.

### Role & Responsibility:

1. **UI/UX Designer**
   - Research.
   - Creating a sitemap and user flow.
   - Designing wireframe (_Low-Fidelity Design_).
   - Designing visual design (_High-Fidelity Design_).
   - Prototyping design.
   - Usability testing.
2. **Front-End Web Developer**
   - Slicing UI design to code.

### Tech Stack & Tools:

- Astro Js
- Bootstrap 5 with custom CSS
- Figma
- VS Code
- GIT

## Project Objectives
    - Build a website that is easy to use and navigate.
    - Showcase a portfolio of attractive digital invitation designs.
    - Provide a simple and efficient ordering process.
    - Enhance the credibility and professionalism of the digital invitation service business.

## Methodology:
    - `Research and analysis`: Understand user needs and preferences, as well as digital invitation market trends.
    - `UI/UX design`: Design wireframes and mockups of the website that focus on ease of use and aesthetics.
    - `Website development`: Implement the UI/UX design into a functional website.
    - `Usability testing and iteration`: Conduct usability testing with users to optimize the website's design and functionality.

## The Process

1. ### Sitemap
      Prioritizing Content Structure with a Sitemap. This project began with the creation of a sitemap, a crucial step that precedes the wireframe development process.  The sitemap served as a foundation for understanding the website's content organization.  By meticulously outlining the website's content structure, I ensured a clear and logical user experience from the outset.

3. ### Low-Fidelity Design (Wireframe)
      Prioritizing Usability with Low-Fidelity Prototyping. In this project, I leveraged low-fidelity prototyping to assess design concepts early in the development cycle. Low-fidelity prototypes are essentially basic representations of the final design, focusing on core elements like layout and functionality rather than visual details. 
      
      This approach proved invaluable for evaluating usability and gathering user feedback on core user flows before investing time and resources in high-fidelity mockups.

4. ### Style Guide
      Establishing Visual Cohesion with a Style Guide. This style guide functioned as a central hub, outlining core visual elements such as typography (including font selection and type scale), color palette, and other design principles.
